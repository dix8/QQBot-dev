# 示例插件

演示所有插件系统能力的示例插件，可自由启用/禁用/删除，作为插件开发的完整参考。

## 指令列表

| 指令 | 说明 | 示例 |
|------|------|------|
| `/ping` | 回复 pong，测试 Bot 是否在线 | `/ping` |
| `/echo <内容>` | 复读你发送的消息内容 | `/echo 你好` |
| `/info` | 查看当前 Bot 的 QQ 号和昵称 | `/info` |
| `/time` | 查看当前时间，并记录调用次数 | `/time` |

## 配置项

在 Web 管理页面点击插件卡片上的「配置」按钮即可修改以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 入群欢迎语 | 文本 | `欢迎加入！` | 新成员加群时 Bot 自动发送的消息 |
| 自动通过好友请求 | 开关 | 关闭 | 开启后收到好友申请会自动同意 |
| 复读模式 | 选择 | 纯文本 | 纯文本：只复读文字；原样转发：保留图片等富文本 |
| 最大复读长度 | 数字 | 100 | 超过此字数的消息 `/echo` 不会复读 |

## 演示的插件能力

- **生命周期**：`onLoad` 加载时读取历史数据，`onUnload` 卸载时记录日志
- **消息处理**：`onMessage` 匹配指令并回复
- **通知处理**：`onNotice` 检测群成员增加事件，发送欢迎消息
- **请求处理**：`onRequest` 根据配置自动通过好友请求
- **API 调用**：`callApi('get_login_info')` 获取 Bot 信息
- **配置读写**：`getConfig` / `setConfig` 读取用户在 Web 页面设置的配置
- **数据存储**：`dataDir` 目录下读写 `counter.json` 持久化调用计数
- **configSchema**：在 `manifest.json` 中声明 4 种配置类型（string/boolean/select/number），Web 页面自动渲染配置表单
