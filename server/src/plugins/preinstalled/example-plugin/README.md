# 示例插件

演示所有插件系统能力的示例插件，可自由启用/禁用/删除，作为插件开发的完整参考。

## 指令列表

### 基础指令

| 指令 | 说明 | 示例 |
|------|------|------|
| `/菜单` / `/menu` | 查看示例插件所有可用指令 | `/菜单` |
| `/ping` | 回复 pong，测试 Bot 是否在线 | `/ping` |
| `/echo <内容>` | 复读你发送的消息内容 | `/echo 你好` |
| `/info` | 查看当前 Bot 的 QQ 号和昵称 | `/info` |
| `/time` | 查看当前时间，并记录调用次数 | `/time` |
| `/赞我` / `/赞` / `/点赞` | 给你的名片点赞 10 次 | `/赞我` |
| `/签到` | 每日签到（仅群聊） | `/签到` |
| `/签到排行` | 查看本群签到排行 Top10 | `/签到排行` |

## 配置项

在 Web 管理页面点击插件卡片上的「配置」按钮即可修改以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 群管理指令 | 开关 | 开启 | 启用群管理相关指令（禁言/解禁/踢/全员禁言/群名片/群公告/改群名/设管理/头衔/退群/撤回/群历史） |
| 入群欢迎 | 开关 | 开启 | 新成员入群时是否发送欢迎消息 |
| 入群欢迎语 | 文本 | `欢迎加入！` | 新成员加群时 Bot 自动发送的消息 |
| 退群通知 | 开关 | 开启 | 成员退群或被踢时发送通知 |
| 防撤回 | 开关 | 关闭 | 开启后消息被撤回时自动展示原始内容（群聊合并转发，私聊直接发送） |
| 管理员变动通知 | 开关 | 开启 | 群管理员设置或取消时发送通知 |
| 禁言通知 | 开关 | 开启 | 群成员被禁言或解禁时发送通知 |
| 新好友欢迎 | 开关 | 开启 | 有新好友添加时自动发送私聊欢迎消息 |
| 新好友欢迎语 | 文本 | `你好！很高兴认识你~` | 新好友添加后发送的消息 |
| 群文件上传通知 | 开关 | 关闭 | 有人上传群文件时发送通知 |
| 荣誉变更通知 | 开关 | 开启 | 龙王、群聊之火等荣誉变更时发送通知 |
| 戳一戳回应 | 开关 | 开启 | 被戳一戳时自动回复 |
| 戳一戳回复语 | 文本 | 多条（`\|` 分隔） | 被戳时随机选取一条回复 |
| 复读检测 | 开关 | 关闭 | 群内连续相同消息达到阈值时自动跟读一次 |
| 复读阈值 | 数字 | 3 | 连续相同消息达到此数量时触发复读 |
| 每日签到 | 开关 | 关闭 | 开启 /签到 指令，每日限一次 |
| 定时消息 | 开关 | 关闭 | 按配置的时间和群号定时发送消息 |
| 定时消息配置 | 文本 | `[]` | JSON 数组，格式见下方说明 |
| 复读模式 | 选择 | 纯文本 | 纯文本：只复读文字；原样转发：保留图片等富文本 |
| 最大复读长度 | 数字 | 100 | 超过此字数的消息 `/echo` 不会复读 |

### 定时消息配置格式

JSON 数组，支持两种格式混用：

**单条定时消息** — 指定 `hour`（0-23）、`minute`（0-59）、`group_id`（群号）、`message`（消息内容）：

```json
[
  {"hour": 8, "minute": 0, "group_id": 123456789, "message": "早上好！"},
  {"hour": 22, "minute": 30, "group_id": 123456789, "message": "该睡觉了~"}
]
```

**整点报时** — 设置 `"type": "hourly"`，用 `messages` 数组按小时 0-23 依次提供消息，可选 `minute`（默认 0）：

```json
[
  {
    "type": "hourly",
    "group_id": 123456789,
    "messages": [
      "🌙 零点已至，晚安好梦",
      "🌙 一点钟，夜色温柔",
      "🌙 两点钟，愿你好眠",
      "...共 24 条，索引 0-23 对应 0-23 点"
    ]
  }
]
```

## 演示的插件能力

- **生命周期**：`onLoad` 加载时读取历史数据和签到数据，`onUnload` 卸载时记录日志、清理内存状态
- **消息处理**：`onMessage` 匹配指令并回复，复读检测追踪连续相同消息
- **通知处理**：`onNotice` 覆盖全部 OneBot V11 通知事件（群成员增减、管理员变动、禁言、撤回、戳一戳、荣誉变更、群文件上传、好友添加）
- **API 调用**：覆盖 OneBot V11 标准 API 及 NapCat 扩展 API（群管理、信息查询、消息操作、群文件）
- **配置读写**：`getConfig` / `setConfig` 读取用户在 Web 页面设置的配置
- **Bot 配置读取**：`getBotConfig('basic')` 读取 Bot 基础配置（如 masterQQ 列表），`getBotConfig('superAdmins')` 读取超级管理员列表，实现权限检查
- **数据存储**：`dataDir` 目录下读写 `counter.json`（调用计数）和 `checkin.json`（签到数据）
- **托管定时器**：`ctx.setInterval` 创建心跳任务和定时消息检查器，卸载时系统自动清除
- **configSchema**：在 `manifest.json` 中声明多种配置类型（string/boolean/select/number），Web 页面自动渲染配置表单
